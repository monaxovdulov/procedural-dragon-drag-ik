# Индекс задач (для AI-агента)

Цель: подготовить понятный и однозначный набор инструкций, чтобы агент мог внести улучшения в `main.py` по готовым планам.

## Что читать и в каком порядке

1. `AGENTS.md` (как запускать проект через `uv`)
2. `docs.md` (как устроен алгоритм и структура `main.py`)
3. `task1.md` (лапки: больше и плавнее)
4. `task2.md` (runtime-ползунки)
5. `task3.md` (пресеты/"скины" параметров)

## Рекомендуемый порядок внедрения

1. Выполнить `task1.md`, затем проверить вручную (`uv run main.py`).
2. Выполнить `task2.md`, затем проверить UI (TAB, drag) и поведение.
3. Выполнить `task3.md`, затем проверить переключение пресетов (F1..F7).

## Важные зависимости между задачами

- Если одновременно реализованы ползунки (`task2.md`) и пресеты (`task3.md`), нужно избежать конфликта:
  `apply_sliders()` (каждый кадр) не должен сразу "перезатирать" параметры, которые выставил пресет.
  Самый простой вариант: при применении пресета обновлять значения `sliders[*].value` (и считать UI источником правды).
- Структурные параметры (`N`, `L`, `leg_pairs`) меняются безопасно только вместе с `dragon.reset()`.
